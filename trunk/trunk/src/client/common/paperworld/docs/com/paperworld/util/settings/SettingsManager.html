Packagecom.paperworld.util.settings
Classpublic class SettingsManager
ImplementsPersistenceManager

SettingsManager. Listeners and settings should be added independently. Should use default values initially in case settings are not loaded. Nothing should be dependent on settings to be retrieved to function. For example should not make the Core in the initialisation process wait for settings to be received from the server before continuing to a lobby or game. A listener will most likely observe more than one setting. Setting instance updates will inform the listener of the specific setting that has changed to avoid unecessary processing. The settings are grouped by group the group they belong too. Therefore the 'settings' hashtable keys are group names and the values are hashtables of settings.
