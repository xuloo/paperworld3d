AbstractUserInput — Class in package com.paperworld.rpc.input

Implements the IUserInput interface.

Initialises two instances of InputData - one for the current timestep and one for the previous timestep, this is required so that the user's current input can be compared with what they did last timestep, if there's a change in behaviour then the input is sent to the server.

Avatar — Class in package com.paperworld.rpc.objects

Avatar handles syncing an object on a client with it's duplicate on the server.

By keeping a set of 3 objects the Avatar can handle keeping in sync with the server while smoothing out any jitter in the receipt of updates from the server.

  • Proxy - is a direct representation of the data recieved from the server.
